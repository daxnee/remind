공공체육시설 위치 안내 서비스 (2022.06.28 ~ 2022.07.11)

개요
팀프로젝트인 건강하슈는 대전시의 공공체육시설의 위치와 정보를 확인할 수 있는 프로그램입니다.
공공체육시설을 일일히 검색해야 하는 번거로움을 개선하고자 프로그램을 제작했습니다.

개발 인원은 총 3명이고 팀원 전원이 백엔드, 프론트를 담당하여 사이트를 제작했습니다.
그 중 지도 페이지의 api 데이터 관리와 지도 폴리곤 구현을 중점적으로 담당했습니다.

건강하슈 프로그램은 공공체육시설의 위치 정보를 보여주는 지도 서비스와 이 것을 관리하는 관리자 페이지로 크게 두 가지 기능이 있습니다.

경험 정리

• 공공데이터 Open API를 사용해 공공체육시설 위치 데이터를 지도 위에 표시했습니다.

• geojson을 이용한 폴리곤 그리기, geocoder를 이용한 주소 위도, 경도 변환, geolocation을 이용한 현재 위치 가져오기 기능을 구현했습니다.

• 디비버를 이용해 엑셀 데이터를 insert 했습니다.

• KAKAO API를 이용해 지도와 마커를 배열에 담고 지우는 기능을 구현했습니다.

• 특정 주소에 해당하는 길찾기와 로드뷰 링크를 마크업에 표시하는 기능을 구현했습니다.

• AWS RDS로 원격 DB를 만들고 EC2로 배포했습니다.

※ 해당 프로젝트는 건강하슈 홈페이지에서 확인할 수 있으며(관리자 계정 : 양다은, 123 / 일반 계정 : 마동석, 123 / 정지 계정 : 정지회원, 123),
프로젝트 소스는 건강하슈 깃허브에서 확인할 수 있습니다.
느낀점
• git으로 프로젝트를 관리하며 개인프로젝트때 느끼지 못한 commit merge 이슈를 경험했으며 pull, commit, push의 순서를 이해했습니다.

• 팀원간 소통의 중요성을 느꼈습니다. 어려운 기능 구현과 잦은 오류 발생시 서로 다양한 의견을 제시했을 때 더 신속하게 문제를 해결 할 수 있었습니다.

• 데이터마다 비교 가능한 유형이 있다는 점을 알게 됐습니다(JSTL에서 가져온 데이터와 Json은 비교 불가능)
더불어 기존 데이터 형식을 다른 형식으로 변환이 가능한 점도 배웠습니다.

• 여러 언어들을 혼용해서 사용할 수 있는 장점과 실무에서는 Jquery와 JSTL을 혼용해서 사용하지 않는다는 언어 사용 방법을 알게 됐습니다.
보안점
• Spring Boot의 DB데이터를 다룰 수 있는 라이브러리를 완벽히 숙지하지 못한 점이 아쉽습니다. 다음 프로젝트에선 이 점을 보충해 효율적인 개발을 하는 것이 목표입니다.

• JavaScript와 JSTL, AJAX 등 여러 언어를 사용하는 탓에 간결한 코드를 작성하는 것에 어려움이 있었습니다. 이 점을 보완해 깔끔한 코드를 구현하고 싶습니다.

• 현재 위치를 기반으로 가까운 체육시설을 표시하는 기능을 구현하지 못한 것이 아쉽습니다.

• KAKAO API의 MAP UI가 한눈에 보기에 다소 복잡하다고 느꼈습니다. 다음 프로젝트에선 더 깔끔한 UI가 있는 MAP API로 사용 해 보고 싶습니다.

건강하슈 메인페이지
• 메인 페이지에서 로그인을 해야만 지도 페이지로 넘어갈 수 있게 구현했습니다.
대전시 지도 페이지
• 행정구별로 나눈 폴리곤을 클릭하면 행정동 구역으로 이동합니다.
마크업 확인 페이지
• 왼쪽에 위치한 종목 카테고리 버튼을 누르면 해당 시설 위치와 정보를 확인할 수 있습니다.

공공체육시설 데이터 관리 페이지
• 추가할 데이터가 있을 경우 삽입, 수정, 삭제가 가능합니다.
회원 관리 페이지
• 회원 계정에 부여된 등급별로(1 : 일반회원/ 2 : 관리자계정 / 3 : 정지회원계정) 접근 가능한 범위를 지정했습니다.
개발 일정 기능명세서 작성
• 프로젝트 관련 사항을 백로그로 작성하여 필요한 기능들을 정리했습니다.
NOTION 사용
• 프로젝트 관리 툴인 NOTION을 사용하여 개발 일정을 체계적으로 기록, 관리했습니다.
